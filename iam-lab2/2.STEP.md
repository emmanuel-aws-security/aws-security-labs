# IAM Lab 1 — Permission Testing Steps

## Step 1: IAM User Setup
- Logged in as IAM user with S3 read-only access

## Step 2: Access S3 Bucket
- Successfully listed and viewed S3 bucket contents

## Step 3: Attempt Unauthorized Actions
Tried the following actions:
- Upload object to S3 bucket
- Delete object from S3 bucket
- Create new S3 bucket

## Step 4: Observe Results
- All write actions were denied by AWS
- Only read actions were allowed

## Step 5: Confirm Behavior
- Verified that permissions strictly control allowed operations
