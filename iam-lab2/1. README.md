# IAM Lab 1 — Permission Testing (S3 Read-Only Behavior)

## 🎯 Objective
To understand how AWS IAM handles denied actions when a user has read-only permissions.

## 🧪 Scenario
An IAM user with S3 read-only access attempts to perform restricted actions such as upload, delete, and bucket creation.

## ☁️ AWS Services Used
- IAM
- S3

## ⚠️ Key Behavior Observed
Actions not explicitly allowed in the IAM policy were automatically denied by AWS.

## 🧠 Key Idea
IAM works on an explicit allow model — anything not allowed is denied by default.
