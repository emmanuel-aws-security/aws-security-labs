## 🧠 Key Learnings

## 🔴 Real Attack Flow
1. Exposure (open SSH / weak config)
2. Access (attacker enters system)
3. Discovery (files, credentials)
4. Exploitation (AWS access, lateral movement)
5. Detection (logs, traces)
6. Remediation (lock down system)

## 🧠 Core Insight
Security failures are rarely one issue—they are chains of misconfigurations.

## ⚠️ Critical Lesson
Hardcoded credentials + open SSH = full account compromise risk.

## 🧾 Final Conclusion
Cloud security is about reducing attack paths, not just deploying infrastructure.
