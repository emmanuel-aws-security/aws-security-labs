# IAM Mini Attack Scenario — Over-Permissioned User Breach Simulation

## 🎯 Objective
To simulate a real-world IAM security failure caused by excessive permissions, then detect and fix it using AWS IAM tools.

## 🧪 Scenario
An IAM user is created with overly broad permissions. The user is able to create and manage AWS resources without restriction.

Later, the misconfiguration is detected and fixed using least privilege principles.

## ☁️ AWS Services Used
- IAM
- EC2
- S3
- IAM Policy Simulator

## 🔥 Security Problem
The IAM user had excessive permissions, allowing unrestricted access to AWS services.

## 🧠 Key Idea
Over-permissioned access creates a full attack surface inside AWS.
