# CloudTrail Lab — Search & Filter Events

## 🎯 Objective
To learn how to properly investigate AWS actions using CloudTrail search and filtering instead of relying only on Event History visibility.

## 🧪 Scenario
Multiple AWS actions are performed, and we investigate them using CloudTrail filtering tools to identify specific user activity.

## ☁️ AWS Services Used
- CloudTrail
- S3
- IAM

## 🔍 Focus
- Event filtering
- Investigation workflow
- Identifying actions by user, event, and resource

## 🧠 Key Idea
Real investigations are not done by scrolling — they are done by filtering and targeting specific events.
