# CloudTrail Incident Investigation — S3 Exposure Activity

## 🎯 Objective
To simulate a suspicious S3 action performed by an IAM user and investigate it using CloudTrail management events.

## 🧪 Scenario
An IAM user performs a risky S3 action, and CloudTrail is used to identify:
- who performed it
- when it happened
- where it came from

## ☁️ AWS Services Used
- CloudTrail
- S3
- IAM

## 🔍 Focus
- Management event investigation
- User attribution
- Security visibility

## 🧠 Key Idea
CloudTrail helps security teams trace AWS account activity back to a specific identity and action.

## ⚠️ Scope
This lab focuses on:
- Management Events
NOT
- S3 Data Events
