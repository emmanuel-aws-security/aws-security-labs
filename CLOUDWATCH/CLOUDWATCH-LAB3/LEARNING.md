# CloudWatch Logs — Learnings

## 🔴 Core Concept
CloudWatch Logs centralizes logs from AWS resources into one location for monitoring and troubleshooting.

## 🧠 What I Learned

### 1. Logs show system behavior
- errors
- service status
- application output

### 2. Centralized logging
Instead of checking each server individually:
- logs are collected in one place (CloudWatch)

### 3. Troubleshooting value
Logs help identify:
- service crashes
- configuration issues
- runtime errors

## 🔍 Key Insight
Without logs, you are guessing when systems fail.

Logs provide real evidence of what happened inside a system.

## ⚠️ Security Importance
Logs can reveal:
- failed access attempts
- system anomalies
- suspicious behavior patterns

## 🧾 Final Conclusion
CloudWatch Logs provide centralized visibility into system and application behavior.
