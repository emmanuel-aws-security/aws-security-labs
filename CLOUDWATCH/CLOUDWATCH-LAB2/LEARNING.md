# CloudWatch Alarm Lab — Learnings

## 🔴 Core Concept
CloudWatch alarms automatically detect unhealthy or abnormal AWS resource behavior.

## 🧠 What I Learned

### CPU Alarm
Used to detect:
- high resource usage
- runaway processes
- possible abuse

### Status Check Alarm
Used to detect:
- instance failure
- networking issues
- AWS infrastructure problems

## 🔍 Key Insight
Monitoring without alerts is incomplete because problems may not be noticed quickly.

## ⚠️ Security Importance
Unexpected CPU spikes may indicate:
- suspicious activity
- compromised workloads
- denial-of-service behavior
- unstable applications

## 🧾 Final Conclusion
CloudWatch alarms improve visibility and early detection of operational or security-related issues.
