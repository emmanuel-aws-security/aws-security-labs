# AWS Security Labs (Hands-On Learning Project)

This repository documents my structured, hands-on learning of AWS security concepts.  
The goal is not just to use AWS services, but to understand how AWS enforces security, access control, and monitoring in real systems.

---

## 🎯 Objective

To build practical understanding of AWS security by breaking down and testing:

- Identity and Access Management (IAM)
- Least privilege principles
- Permission boundaries and policy effects
- Data access security (S3)
- Logging and visibility (CloudTrail)
- Real-world misconfiguration risks

---

## 🧠 Learning Approach

Instead of only reading documentation, I focus on:

- Creating AWS resources from scratch
- Intentionally testing permissions (what works vs what fails)
- Observing AWS enforcement behavior
- Documenting outcomes and security implications
- Breaking things to understand why they break

This helps move from “knowing AWS” → “understanding AWS security behavior”.

---

## 🧪 Lab Structure

Each lab is documented separately with:

- Setup steps
- Actions performed
- Permission behavior observed
- Key security findings
- Real-world implications

## 🔍 Key Focus Areas

- Least privilege design in real scenarios
- How AWS denies unauthorized actions
