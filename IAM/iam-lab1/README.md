# IAM Lab 1 — IAM Basics (Users, Groups, Policies)

## 🎯 Objective
To understand how IAM controls access in AWS using users, groups, and policies without using the root account.

## ☁️ AWS Services Used
- IAM

## 🧪 What I Built
- Created an IAM user (without using root for daily work)
- Created an IAM group
- Attached a policy to the group
- Added the user to the group
- Tested access permissions

## 🧠 Key Idea
IAM access is not given directly in most cases. It is controlled through groups and policies that define what a user can or cannot do.

## ⚠️ Security Focus
Root account should not be used for normal tasks because it has full control over the AWS account with no restrictions.
