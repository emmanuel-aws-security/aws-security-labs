# S3 Lab — Public vs Private Learnings

## 🔴 Core Concept
S3 objects are private by default unless explicitly made public.

## 🧠 Key Insight
Public access is a configuration choice—not automatic.

## 🔍 What Happened
- Private object → access denied
- Public object → accessible via URL

## ⚠️ Security Risk
If sensitive data is made public:
- anyone with the link can access it
- no authentication is required

## 🧠 Real-World Meaning
Many data breaches happen because S3 buckets or objects are accidentally made public.

## 🧾 Final Conclusion
Access control in S3 must be handled carefully to prevent unintended data exposure.
