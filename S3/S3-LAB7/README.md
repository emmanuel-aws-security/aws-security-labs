# S3 Breach Simulation — Public Bucket Leak

## 🎯 Objective
To simulate a real-world S3 data exposure incident and demonstrate detection, investigation, and remediation.

## 🧪 Scenario
A bucket containing sensitive data is accidentally made public, allowing anyone on the internet to access it without authentication.

## ☁️ AWS Service Used
- S3
- IAM

## 🔍 Focus
- Misconfiguration → Exposure
- Investigation → Root cause
- Fix → Least privilege
- Validation → Confirm security

## 🧠 Key Idea
S3 breaches often happen due to simple misconfigurations, not complex attacks.

## ⚠️ Risk
Public buckets expose sensitive company data to the entire internet.
