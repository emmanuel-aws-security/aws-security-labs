# Cloud Security Incident Simulation (Final Project)

## 🎯 Objective
Simulate a realistic AWS cloud security incident involving exposed infrastructure, weak permissions, and insecure configurations.

This project demonstrates:
- attack surface awareness
- investigation workflow
- monitoring visibility
- remediation process
- layered cloud security thinking

---

- EC2
- IAM
- VPC
- S3
- CloudTrail
- CloudWatch

### Public Components

### Private Components
- Private subnet

### Security Components
- IAM roles
- Security groups
- CloudTrail logging
- CloudWatch monitoring

---

## 🔥 Incident Scenario
A cloud environment was intentionally deployed with weak security controls to simulate realistic exposure risks and investigation workflows.

The project focused on:
- exposed SSH access
- insecure S3 access
- excessive permissions
- monitoring and detection
- remediation and validation

## ⚠️ Key Security Insight
Most cloud breaches are caused by:
- misconfiguration
- overexposure
- poor visibility
NOT advanced hacking.
