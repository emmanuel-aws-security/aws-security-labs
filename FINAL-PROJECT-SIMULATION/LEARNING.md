# Final Learnings

## 🧠 Major Lessons Learned

### 1. Cloud Security Is Layered
Failures can occur in:
- compute
- network
- identity
- monitoring
- storage

---

### 2. Misconfiguration Causes Most Risk
Examples:
- open SSH
- public S3 buckets
- excessive permissions

---

### 3. Monitoring & Logging Matter
Without:
- CloudTrail
- CloudWatch
- logs

investigation becomes difficult.

---

### 4. Troubleshooting Requires Structure
Useful debugging flow:
1. Check service
2. Check ports
3. Check local access
4. Check network rules
5. Check logs
6. Check IAM

---

## 🧾 Final Conclusion
This project demonstrated:
- infrastructure deployment
- attack simulation
- investigation workflow
- remediation process
- cloud security thinking
